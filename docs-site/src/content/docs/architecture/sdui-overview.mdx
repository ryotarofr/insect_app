## 三層の責務 (要約)

`Region` (= ページ内の意味的領域) → `Block` (= 最小単位の意味的部品) → `Role` (= ブロック内の役割) という階層で UI を server-driven に表現します。各層が「何を表現するか」「何を表現しないか」を明確に分けることで、テンプレート (= `CardBlock` の variant) を増やしても破綻しない契約を維持します。
