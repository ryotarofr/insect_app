モバイル設計の要点

  • サイドバー → ボトムタブ: デスクトップの 8 項目を 5 タブ（ホーム / 探す / 出品+ / 飼育 / マイ出品）に集約。 中央の ＋ は既存 QuickLogFab を昇格させて Action Sheet を出す（出品 / 個体登録 / 群作成 / 飼育ログ）。
  • KPI は横スクロール 5 枚。デスクトップ 6 列を 1 列に潰すと縦が伸びすぎるので、 snap 付き horizontal scroll で「飼育系（青）→ 販売系（緑）」の順に並べる。
  • マイ出品サマリも横スクロール。ホーム上はカードで概要、詳細は「すべて見る → /listings/me」で 縦リスト化したフルページに遷移。
  • 出品作成は 4 ステップウィザード。デスクトップの 2 カラムを縦展開すると視認性が悪化するので、 モバイルは 個体選択 → 写真+説明 → 価格+配送 → 確認 に分割。 進捗は上端の 4 セグメントバーで表現（既存ステッパーと同じ視覚言語）。
  • sticky bottom CTA はモーダル系画面（出品作成）専用。ボトムタブとは併存させず、 モバイル native の「画面外に CTA を逃がさない」原則を踏む。
  • 確認画面は accordion。価格 / 配送 / 説明を畳み、サマリ値だけ summary に表示。 実装的には <details> の native open/close で十分。
  • SafeArea: 上端は status-bar（36pt）、下端は home-indicator 用に bottom-cta の padding-bottom を 18px に。 実機では env(safe-area-inset-bottom) で動的に調整。
  • BottomTabBar 既存実装の拡張: 現状 BottomTabBar.tsx は飼育系のみ想定なので、 マーケット系（探す / マイ出品）を 2 タブ追加し、中央 FAB スロットを差し込む構造にリファクタが必要。
