このモックの設計メモ

  • 2 カラム + sticky 右パネル。現状の /listings/new 構造を踏襲しつつ、左に個体ピッカー、 写真 + 説明、右にフォーム + 手数料 + プレビューを配置。スクロール中もアクションが見える。
  • 個体カードに状態タグ。archived は非表示、既出品 は disabled + 出品 ID 表示で誤操作防止。 性別 / ステージは色分けタグで判別性を上げる。
  • 説明文は 3 モード: プレビュー（rendered）/ 編集（プレーン）/ マークダウン。 カルテ自動生成のときは 「下書き：カルテから自動生成」 ラベルを残し、編集後は 「編集済」 に切替。
  • 推奨価格: specimens + orders/order_items から同種同サイズ帯の中央値を出す。 将来の AI 価格推定（Phase 6 出品自動アシスト）の差し込み口になる。
  • 手数料サマリは想定落札額ベースで動的計算。販売手数料 10% / Stripe 3.6% は features.md の C2C 経済設計に準拠。
  • 出品ボタンは前提条件で disable。Stripe Connect 未連携 / 写真 0 枚 / タイトル空 / 価格 0 は ボタン無効 + チェックリストで原因表示（accountability）。
  • API: POST /api/v1/listings と整合 （specimen_id, title, description, is_auction, starting_price_jpy, ends_at）。配送方法は別 endpoint PATCH /listings/{id}/shipping_methods を新設するか、出品時に listings に allowed_shipping_methods JSONB を追加する余地。
  • 下書き保存: listings.status='draft' を CHECK 制約に追加するマイグレーションが必要。 現状は active / sold / canceled / expired のみ。
