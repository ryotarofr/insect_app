このモックの設計メモ

  • マイページ＝自分専用ハブ。「飼育の活動」と「販売の活動」を 1 画面で俯瞰できる。 KPI は左 3 枚を 飼育系、右 3 枚を 販売系 に色分け（青タグ／緑タグ）。
  • 詳細 CRUD は別ルート。マイページ §02 はサマリ＋直近 3 件のみで、フッターから /listings/me（= マイ出品ページ）に逃がす。specimens/me・cohorts/me・orders/me という既存パターンに揃える。
  • サイドバー再編: マーケット配下に「マイ出品」「取引履歴」を新設。/products は引き続き「他人の出品閲覧」（公開市場）として残し、自分のものとはルートで分離する。
  • API 提案: GET /api/v1/listings/me?status=active|sold|canceled を新設し、 v_listings_with_counts を seller_user_id = me でフィルタする。 タブごとの件数は SQL 側で GROUP BY status。
  • 取引タイムライン: 売却 (+¥) と購入 (−¥) を 1 リストに混ぜる。実装的には orders WHERE buyer = me ∪ orders WHERE seller = me（後者は今未実装）を UNION ALL + created_at DESC。
  • Stripe Connect バナーは state ベースで出し分け（連携済 / 未連携 / 入金待ち）。 他にも「死着補償の対応待ち」「未受領評価あり」などサイト横断アラートのスロットとしても使える。
  • ヒーローのアクション: 既存の「+ 新規 ▾」ドロップダウンを 3 ボタン平置きに展開。 C2C 化で「+ 出品する」をプライマリに昇格。
