個体カルテ＋飼育ログ の再設計

現状は V1〜V5 のバリアント併存と、記録フォーム／タイムラインの2カラム構成が、 ユーザーに「どこを見ればよいか」の判断を強いています。 本提案では、1ヒーロー＋3タブ の個体カルテと、 Quick Log バー中心の飼育ログに整理し、初心者にも熟練ブリーダーにも自然に使える情報設計に統合します。

設計原則 ① 個体カルテ ② 飼育ログ Before / After サマリ
§00

設計原則 / 3 PRINCIPLES

WHY this redesign
01 / ONE HERO

「今、何が大事か」を一目で。

縦積みのカード羅列をやめ、最新写真・ステージ・次の羽化までの日数・体重差分を ひとつのヒーロー領域に集約。スクロールしなくても「状態」が読めることを最優先にします。

02 / PROGRESSIVE DISCLOSURE

詳細はタブで段階的に。

基本情報・ログ・血統を 3つのタブに分割。初心者には見せすぎない、 ブリーダーには深く辿れる ── を「同じ画面」で両立します。5つのレイアウト切替は廃止。

03 / LOG IN 3 TAPS

記録は常に3タップ以内。

飼育ログは対象個体があらかじめ選ばれた状態で Quick Log バーを常時提示。 「種別 → 値 → 保存」で完了。PCでも片手スマホでも摩擦なく記録できます。

§03

Before / After サマリ

要点の差分
観点Before（現状）After（本提案）
CARTE LAYOUT V1〜V5 の5バリアント併存。画面右上で切替 ─ 意思決定が曖昧で、ユーザー・開発どちらにも負荷。 1ヒーロー + 3タブ（概要 / ログ / 血統）に統合。バリアント切替廃止。
CARTE HERO 写真・ステージバー・直近ログ・基本情報カード・血統カード・メモ・羽化予測が縦に積層。 写真＋KPI3点（体重 / サイズ / 次の羽化）＋主要アクション1つをヒーロー化し、"状態"が一目で読める。
MEMO 保存ボタン必須・dirty判定・成功トースト ─ 摩擦あり。 自動保存＋「● 自動保存」インジケータ。最終更新時刻のみ控えめに表示。
LOG 追加導線 カルテから「＋ログ追加」→ 別ページへ → 対象個体をSELECTで再選択。 カルテから起動 → 対象個体プリセット済みのモーダル／シートでその場記録。
LOG PAGE 左360pxの固定フォーム + 右カラムのタイムライン。フォームが常に場所を占有。 上部 Quick Log バー（折り畳み可） + 1カラムタイムライン。閲覧時は画面幅をフル活用。
LOG TYPES 体重・給餌・マット・脱皮・観察を等価な2x2グリッドで提示。 利用頻度順（体重→給餌→観察→脱皮→マット）の水平chip。主要3つが手前に来る。
FILTERS 種別のみ1軸。全個体横断で混ざる。 種別 / 個体 / 期間 の3軸セグメント。デフォルトで直近30日に絞り、認知負荷を軽減。
MOBILE デスクトップ2カラム前提。片手スマホで入力しづらい。 下部FAB → ボトムシート、種別chipスクロール、3タップで記録完了。
BRAND TONE 博物誌×カルテ風の作り込みは魅力だが、装飾ラベル（SPECIMEN CARTE, BLOODLINE...）が過剰で本文より目立つ場面あり。 装飾ラベルは小さな mono-label に統一。セクションタイトル（serif）で階層化し、ブランドトーンは維持。
